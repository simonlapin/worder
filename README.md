# Worder

Личное iOS-приложение для заучивания английских слов (EN ↔ RU) с интервальными повторениями (FSRS). Один пользователь, один iPhone, все данные локально.

## Структура

- `WorderCore/` — SPM-пакет с логикой (схема данных, планировщик, проверка ответов, импорт). Тестируется на macOS без симулятора.
- `App/` — SwiftUI-оболочка (iOS 26+).
- `AppTests/` — тесты уровня приложения.
- `data/` — эталонные пачки слов в каноническом JSON.
- `scripts/` — разовые инструменты (конвертация PDF → JSON).
- `project.yml` — источник правды для Xcode-проекта; `Worder.xcodeproj` генерируется и не коммитится.

## Команды

```sh
# Тесты ядра (быстрые, без симулятора)
swift test --package-path WorderCore

# Сгенерировать Xcode-проект (после изменения project.yml)
xcodegen generate

# Сборка приложения
xcodebuild build -project Worder.xcodeproj -scheme Worder \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro,OS=26.1'

# Тесты приложения
xcodebuild test -project Worder.xcodeproj -scheme Worder \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro,OS=26.1'
```

Разработка в Xcode: `xcodegen generate && open Worder.xcodeproj`.

```sh
# Release-сборка (проверка перед установкой на устройство)
xcodebuild build -project Worder.xcodeproj -scheme Worder -configuration Release \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro,OS=26.1'
```

## Установка на iPhone

1. `xcodegen generate && open Worder.xcodeproj`.
2. Подключите iPhone кабелем (или по Wi-Fi, если включена беспроводная отладка) и выберите его как destination.
3. Подпись настроена в `project.yml`: `CODE_SIGN_STYLE: Automatic`, `DEVELOPMENT_TEAM: 7427PD678R`. Для другой команды поменяйте `DEVELOPMENT_TEAM` и перегенерируйте проект.
4. Нажмите Run. При первой установке подтвердите доверие разработчику на устройстве: Настройки → Основные → VPN и управление устройством.
5. Если iOS на устройстве новее SDK Xcode (например, устройство 26.5 при SDK 26.1), Xcode может попросить скачать файлы поддержки устройства — согласитесь; если сборка не встаёт, обновите Xcode.

Слова импортируются автоматически при первом запуске из встроенной пачки `core-1500`. Без сети и без API-ключа приложение полностью функционально.

Два режима занятий с главного экрана:

- **Заниматься** — обычная сессия: повторения по расписанию FSRS плюс новые слова до дневного лимита.
- **Свободная тренировка** — весь словарь в произвольном порядке, доступна всегда и без ограничений. Чистый тренажёр: ответы попадают в историю и статистику, но не двигают расписание повторений и не расходуют дневной лимит новых слов.

## Настройки приложения

- **Новых слов в день** — лимит ввода новых слов в обычной сессии (пресеты от 0 до 500 и «Без лимита», по умолчанию 20). Без лимита все нетронутые слова подмешиваются сразу — объём ежедневных повторений быстро вырастет.
- **Напоминания** — ежедневные локальные уведомления в заданные времена; текст включает число слов к повторению. При первом включении iOS спросит разрешение на уведомления.
- **Anthropic API (опционально)** — вставьте ключ (`sk-ant-…`) в Настройки → Anthropic API. Ключ хранится только в Keychain устройства, не логируется и не попадает в резервную копию. С ключом приложение фоново генерирует предложения-примеры (включается упражнение «контекст») и подсказки для систематически проваливаемых слов.
- **Резервная копия** — экспорт всего состояния (слова, прогресс, история, кеш предложений, настройки — без API-ключа) в один JSON-файл через share sheet и восстановление из него. Импорт в непустую базу запрашивает подтверждение и полностью замещает текущие данные.

## Как добавить новую пачку слов

1. Составьте JSON по схеме ниже (уникальный `batchId`, `id` слов уникальны внутри пачки) и положите в `data/`.
2. Добавьте файл в ресурсы приложения в `project.yml` (рядом с `data/core-1500.json`):
   ```yaml
   - path: data/my-batch.json
     buildPhase: resources
   ```
3. Добавьте импорт пачки в `App/AppBootstrap.swift` (по образцу `core-1500`; импорт идемпотентен, прогресс уже изученных слов не сбрасывается).
4. `xcodegen generate`, пересоберите и установите приложение.

## Формат пачки слов (контракт)

Все пачки слов поставляются в JSON по этой схеме (`schemaVersion: 1`). Эталонный пример — `data/core-1500.json`.

```json
{
  "schemaVersion": 1,
  "batchId": "core-1500",
  "title": "1500 самых употребляемых английских слов",
  "category": null,
  "words": [
    {
      "id": 1,
      "word": "the",
      "translations": ["артикль"],
      "note": "определённый"
    },
    {
      "id": 760,
      "word": "ring",
      "translations": ["кольцо", "звонить"],
      "sentences": [
        { "en": "She wears a gold ring.", "ru": "Она носит золотое кольцо." }
      ]
    }
  ]
}
```

Правила:

- `schemaVersion` — всегда `1`. Другие версии приложение отклонит с понятной ошибкой.
- `batchId` — уникальный идентификатор пачки (латиница/дефисы). Повторный импорт пачки с тем же `batchId` идемпотентен: слова не дублируются, прогресс не сбрасывается.
- `id` — целое, уникальное внутри пачки. Пара `(batchId, id)` — стабильный ключ слова.
- `word` — английское слово, непустое.
- `translations` — список **отдельных значений** (уже разделённых), каждое непустое. Не склеивать значения в одну строку через запятую.
- `note` — необязательная грамматическая пометка («определённый», «без сущ.»), не является переводом.
- `category` — необязательная тематическая категория (у пачки и/или у слова); используется для подбора дистракторов и статистики.
- `sentences` — необязательные примеры: пары `en`/`ru`, обе стороны непустые; предложение должно содержать слово в точной форме. Внимание: в текущей версии импортёр эти примеры не переносит в кеш — примеры генерирует приложение само (при наличии API-ключа).
